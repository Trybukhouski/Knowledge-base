**HTTP (Hypertext Transfer Protocol)** — это протокол прикладного уровня на базе TCP/IP, который является фундаментом Всемирной паутины и используется для загрузки веб-страниц с помощью гиперссылок.

Вот основные технические принципы протокола, описанные в источниках:

- **Модель взаимодействия:** Работа строится по принципу «запрос-ответ». Клиент (обычно браузер) инициирует запрос к серверу, который в ответ отправляет сообщение с результатом.
- **Сетевое расположение:** HTTP — это уровень абстракции (7-й уровень модели OSI), который стандартизирует общение хостов, полагаясь на TCP/IP для физической доставки данных. По умолчанию используется порт 80, а для защищенного HTTPS — порт 443.
- **Отсутствие состояния (Stateless):** Протокол является «безстатусным», то есть сервер не сохраняет информацию о клиенте между запросами. Каждый запрос должен содержать все данные, необходимые для его выполнения.
- **Структура сообщений:**
    - **Запрос** включает версию HTTP, URL, метод (например, GET или POST), заголовки и необязательное тело сообщения.
    - **Ответ** содержит код состояния (например, 200 OK или 404 Not Found), заголовки и тело (например, HTML-код страницы).
- **Эволюция:** Протокол прошел путь от версии 0.9 (1991 год), поддерживавшей только метод GET, до современных HTTP/2 и HTTP/3, которые используют бинарные форматы и новые транспортные протоколы (QUIC) для минимизации задержек.
