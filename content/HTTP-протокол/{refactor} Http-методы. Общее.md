Методы HTTP-запроса (которые часто называют **«глаголами»**) представляют собой стандартные команды, указывающие серверу, какое именно действие клиент (например, браузер) хочет выполнить с конкретным ресурсом.

### **Общая информация о методах HTTP-запроса**

- **Назначение:** Метод определяет характер взаимодействия. Он сообщает серверу, нужно ли просто передать данные клиенту, принять новую информацию или изменить существующий файл.
- **Место в структуре:** В текстовом формате протокола (HTTP/1.x) метод всегда стоит в самом начале первой строки запроса. В бинарном протоколе **HTTP/2** метод передается в виде специального **псевдо-заголовка `:method`**.
- **Эволюция:** Количество методов росло вместе с развитием интернета. Если в самой первой версии **HTTP/0.9 (1991 год)** существовал только один метод, то в версии **HTTP/1.1** их стало значительно больше, включая специализированные команды для управления ресурсами.
- **Связь с концепцией Stateless:** Поскольку HTTP является протоколом **без сохранения состояния**, сервер не «помнит» предыдущие запросы. Поэтому каждый метод должен использоваться в контексте запроса, который содержит абсолютно всю информацию, необходимую для выполнения действия.
- **Основные методы (согласно HTTP/1.1):** Кроме GET и POST, существуют **PUT** и **PATCH** (для обновления данных), **DELETE** (для удаления), **HEAD** (запрос только заголовков без тела) и **OPTIONS** (проверка возможностей сервера).
