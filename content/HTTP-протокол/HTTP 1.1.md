#atom #linking 
> [!abstract] **Главное** 
> Стандарт, доминировавший в вебе более 15 лет. Его главной миссией стало исправление проблем производительности HTTP/1.0: 
> - Соединения стали **постоянными по умолчанию**, что позволило передавать множество файлов через один открытый канал. 
> - Также версия ввела обязательный заголовок **Host**, позволивший размещать сотни разных сайтов на одном IP-адресе, и расширила набор команд для более гибкого управления ресурсами (PUT, DELETE и др.). ^b3c57d
## Соединения в HTTP 1.1 стали устойчивыми
HTTP/1.1 кардинально изменил механику работы с сетью, чтобы минимизировать задержки:
- **Persistence по умолчанию:** Больше не нужно открывать новое соединение для каждого изображения; браузер использует одно и то же «рукопожатие» для серии запросов.
- **Pipelining (Конвейер):** Теоретически клиент мог отправить пачку запросов, не дожидаясь ответа на первый. Однако на практике это часто приводило к **блокировке очереди (Head-of-Line blocking)**, когда один «тяжелый» ответ тормозил все остальные, идущие за ним.
## В HTTP 1.1 стала важна детерминированность границ
Поскольку соединение больше не разрывается после каждого файла, возникла необходимость четко определять границы ответов в общем потоке данных:
- **Заголовок Content-Length:** Заголовок указывает точный размер тела ответа в байтах. Это необходимо, чтобы клиент понимал, где заканчивается один ответ и начинается другой в условиях постоянного соединения.
- **Потоковая передача (Chunked Encoding):** Если сервер не знает итоговый размер данных заранее (динамический контент), он отправляет тело по частям (чанками). Каждая часть имеет свой размер, а завершается передача пустым чанком с нулевой длиной.
## В HTTP 1.1 появился виртуальный хостинг и заголовок Host
В этой версии заголовок `Host` стал обязательным.
>**Зачем это нужно**
>Раньше сервер определял сайт по IP-адресу. С введением `Host` клиент явно сообщает имя домена, что позволило провайдерам экономно использовать IP-адреса, размещая множество сайтов на одном сервере.
## HTTP 1.1 получил расширенный инструментарий в виде методов
Набор команд стал соответствовать полноценной системе управления ресурсами:
- **PUT и PATCH:** Для загрузки и частичного обновления файлов.
- **DELETE:** Для удаления ресурсов.
- **OPTIONS:** Для проверки возможностей сервера.
