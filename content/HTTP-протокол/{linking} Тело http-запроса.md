## Концепция и Назначение

Тело http-запроса — это часть http-запроса, которая содержит то, **что** передавать, в то же время как заголовки описывают то, **как** передавать.
Тело используется опционально. То есть если данных для передачи нет (например, переход по ссылке), тело отсутствует.
### Что можно передавать
 - Текст из полей ввода
 - логины/пароли
 - JSON-объекты для API
 - бинарные файлы (фото/видео)
 - др...
## Связь с методами
Использование тела диктуется логикой HTTP-глаголов:
- **Активные (POST, PUT, PATCH):** Создают или обновляют данные, поэтому почти всегда несут тело.
- **Пассивные (GET, DELETE):** Запрашивают или удаляют по идентификатору. Данные передаются в URL, тело правилами «хорошего тона» не предусмотрено.
## Техническая эволюция 
Способ упаковки данных менялся вместе с версиями протокола:
- **HTTP/1.1:** Текстовый поток. Тело отделяется от заголовков **пустой строкой**. Важен заголовок `Content-Length`, чтобы сервер знал, где конец данных.
- **HTTP/2:** Бинарный формат. Данные упакованы в **DATA frames**. Это позволяет передавать тело асинхронно, не блокируя другие запросы.
- **HTTP/3:** Пакеты QUIC. Тело нарезается на части, которые летят максимально быстро и защищены шифрованием «из коробки».
## Безопасность
Тело запроса — самое уязвимое место, так как там часто лежат пароли.
- **Проблема:** В обычном HTTP тело передается открытым текстом.
- **Решение:** Использование **порта 443** и **TLS-шифрования**. В этом случае всё тело «упаковывается» в зашифрованный туннель, невидимый для посредников.
