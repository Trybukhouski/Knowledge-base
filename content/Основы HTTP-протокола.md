**HTTP (Hypertext Transfer Protocol)** — это прикладной протокол передачи данных, который является фундаментом Всемирной паутины. Он стандартизирует способ взаимодействия между клиентами (например, браузерами) и серверами, определяя правила запроса и передачи контента в интернете.

Вот основные технические и концептуальные основы протокола:

- **Работа протокола строится по принципу «запрос-ответ».** 
  Клиент отправляет [[http-запрос]], содержащий метод, определяющий действие, URL, версию протокола и заголовки, передающий служебную информацию, а сервер возвращает [[http-ответ]] со статусом выполнения, представляющий собой код состояния, и запрошенными данными.
- **HTTP работает на прикладном уровне (Layer 7 модели OSI)**
  Также обычно использует протокол TCP для обеспечения надежной доставки данных. По умолчанию для незащищенного соединения используется порт 80, а для защищенного (HTTPS) — порт 443.
- **Это stateless-протокол**
  Это означает, что сервер не хранит информацию о предыдущих запросах клиента — каждый запрос обрабатывается независимо и должен содержать всю необходимую информацию для его выполнения.
