HTTP-методы (их часто называют «глаголами») — это фундаментальная часть HTTP-запроса, которая указывает серверу, какое именно действие необходимо выполнить с конкретным ресурсом.

Вот основные характеристики методов на абстрактном уровне:

*   **Назначение:** Методы стандартизируют общение между клиентом и сервером. Каждый запрос сообщает серверу об ожидаемом результате: хотите ли вы просто получить информацию, отправить новые данные или изменить существующий ресурс.
*   **Эволюция и расширение:** Список методов рос вместе с развитием интернета. В самой первой версии (HTTP/0.9) существовал только метод `GET`. С версией 1.0 появились `POST` и `HEAD`, а версия 1.1 добавила инструменты для полноценного управления данными: `PUT`, `PATCH`, `DELETE` и `OPTIONS`.
*   **Связь с телом запроса:** Выбранный метод определяет структуру сообщения. Например, методы для отправки данных (такие как `POST`) обычно включают в себя «тело» (body) с полезной нагрузкой, в то время как методы для простого получения информации (как `GET`) чаще всего тела не имеют.
*   **Место в структуре:** Метод всегда находится в первой строке запроса. В современной версии HTTP/2 методы для эффективности передаются не текстом, а в виде специальных бинарных псевдо-заголовков (например, `:method`).
*   **Обработка:** Сервер анализирует метод, чтобы понять, какие ресурсы выделить для обработки запроса. В продвинутых версиях протокола клиент может даже назначать приоритеты разным методам, чтобы важные действия выполнялись быстрее.
