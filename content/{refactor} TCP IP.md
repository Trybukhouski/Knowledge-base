**TCP/IP** — это базовый стек сетевых протоколов, на котором построен весь современный интернет и протокол HTTP в частности. Если HTTP определяет, _что_ именно мы передаем, то TCP/IP отвечает за то, _как_ эти данные физически доставляются между устройствами.

Вот основные факты о нем:

- **Надежность и порядок:** Протокол TCP (часть стека) гарантирует, что информация будет доставлена полностью и пакеты данных соберутся в правильной последовательности.
- **Трехэтапное рукопожатие (Three-way handshake):** Перед началом передачи данных клиент и сервер обмениваются тремя служебными пакетами (**SYN**, **SYN-ACK**, **ACK**), чтобы подтвердить готовность обеих сторон к связи.
- **Порты:** Для идентификации трафика TCP использует порты. По умолчанию для HTTP зарезервирован порт 80, а для защищенного HTTPS — порт 443.
- **Влияние на скорость:** Установление нового TCP-соединения — это ресурсозатратный процесс из-за необходимости «рукопожатия». Именно поэтому в версиях HTTP/1.1 и выше появились «постоянные соединения», позволяющие отправлять несколько запросов через одно уже открытое TCP-соединение.
